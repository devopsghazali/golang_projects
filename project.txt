  > npm install motion lucide-react
      - motion       : Framer Motion (animations)
      - lucide-react : clean modern icons
  > npm install -D tailwindcss @tailwindcss/vite
      - Tailwind CSS v4 (latest, with Vite plugin)

STEP 3 — Vite + Tailwind Setup:
  - vite.config.js me tailwindcss() plugin add kiya.
  - src/index.css me Tailwind v4 import + custom theme tokens
    (@theme block) + utility classes (glass-card, bg-grid-light,
    bg-grid-dark, mask-fade, text-gradient-brand, shadow tokens).
  - Dark mode: .dark class-based, main.jsx pe system preference
    + localStorage ("dw-theme") read karke initial apply.

STEP 4 — index.html polish:
  - Title, description, theme-color meta, Inter font preload.

STEP 5 — Cleanup:
  - Default Vite scaffold files (App.css, assets/*) delete kiye.

STEP 6 — Assets:
  - public/logo.svg : Custom DarkWorld monogram "D" shape with
    dark gradient background + subtle purple->cyan accent stroke.
  - public/favicon.svg : compact version of the logo.

STEP 7 — Data layer:

STEP 8 — Components banaye (src/components/):

  a) Background.jsx
     - Fixed full-screen, -z-10.

     - 4 blurred gradient "particles" slowly animating (x, y,
       scale loops via Framer Motion).
     - Top + bottom soft gradient overlays for depth.
     - useReducedMotion respect karta hai.

  b) ThemeToggle.jsx
     - Fixed top-right glass pill button.
     - Sun/Moon icon swap with rotate + fade (AnimatePresence).
     - localStorage me theme persist.

  c) ProfileHeader.jsx
     - Logo (96-112px circular) with soft glow halo.
     - Animated green status dot (ping animation).
     - Brand name "DarkWorld Official" (bold, tight tracking).
     - Subtitle with blue->purple->cyan gradient text.
     - 3-line trust-building description.
     - Stagger-fade-in on mount.

  d) TrustBar.jsx
     - 3 micro badges: "Active creator · Verified community ·
       Replies within 24h".
     - Uppercase tracking, emerald dots — whisper-quiet authority.

  e) LinkCard.jsx  (core piece — liquid fill animation)
     - Glassmorphism card: blur + semi-transparent bg + soft
       border + soft shadow + rounded-2xl.
     - Layout: [icon tile] [title + subtitle] [arrow icon].
     - Hover: slight lift (y: -2) + deeper shadow.
     - Click pe:
         * Overlay motion.div clipPath animate:
             inset(0 100% 0 0)  ->  inset(0 0% 0 0)
           (left se right tak smooth liquid fill)
         * Duration ~550ms, ease [0.65, 0, 0.35, 1].
         * onAnimationComplete pe window.open(href, "_blank").
         * Double-click guard (filling state).
     - Fill hote time text + icon white ho jate hain.
     - Keyboard support: Enter / Space bhi trigger karta hai.
     - useReducedMotion respect: animation skip + direct redirect.

  g) Footer.jsx

6. KEY DESIGN DECISIONS (kyu aise banaya)
----------------------------------------------------------------

- clipPath se liquid fill: ye CSS transform se zyada smooth
  dikhta hai, aur per-card color/gradient dono support karta hai
  (solid color + Instagram ka gradient dono kaam karte hain).

- Trust bar deliberately whisper-soft rakha (small uppercase
  tracking, muted color) — loud badges trust kam karte hain,
  quiet ones authority build karte hain.

- Grid background + blurred gradient particles: depth deta hai
  without noise. mask-fade se edges me softly disappear hota hai.

- Tailwind v4 (@theme + @custom-variant): latest idiomatic
  setup, no tailwind.config.js file required.

- motion/react (naya import path): Framer Motion ka current
  packaging — "motion" package use ho raha hai, purana
  "framer-motion" naam nahi.

- Reduced-motion users ke liye animation skip + direct redirect
  (accessibility).

  TABLES (new migration: 20260420_coupon_system.sql)
    a) coupons

    b) coupon_redemptions (ledger)
         coupon_id, coupon_code, purchase_id, customer_email,

    c) coupon_audit_log

  RPC FUNCTIONS (SECURITY DEFINER):
    preview_coupon(code, email, cart_amount) — read-only check
        (used by validate_coupon edge fn). Returns ok + discount
        + remaining_slots. Does NOT reserve.
    try_reserve_coupon(code, email, cart_amount, hold_minutes)
        — race-safe: SELECT FOR UPDATE on coupon row, checks
        all gates, inserts redemption (status=reserved,
        expires_at=now+30min). Returns redemption_id + discount.
    confirm_coupon_reservation(redemption_id, purchase_id)
        — marks reserved → confirmed, bumps used_count,
        idempotent (already_confirmed flag if re-called).
    release_coupon_reservation(redemption_id)
        — reserved → cancelled. Used on checkout failure in
        create_order's catch block.
    max_users ke check me reserved (non-expired) + confirmed
    dono count hote hain, race-safe.

  EDGE FUNCTIONS (new + modified):
    1) validate_coupon (new, public)
         - Rate limit: 20/min per IP (in-memory sliding window)
         - Input: code + courseId + customerEmail (optional)
         - Returns: discountAmount, finalAmount, remainingSlots
         - Pure preview, no side effects.
    2) admin_coupons (new, admin-only)
         - Guard: x-admin-token header must safeEqual
           ADMIN_API_TOKEN env (length check + constant-time).
         - Rate limit: 120/min per IP (brute-force defense).
         - Actions: list / stats / create / update / delete /
           toggle. All writes log to coupon_audit_log.
         - Code validation: /^[A-Z0-9_-]{3,32}$/.
         - Percentage cap 100, max_users must be > 0, etc.
    3) create_order (modified)
         - Accepts optional couponCode.
         - Calls try_reserve_coupon → gets discount → uses
           finalAmount (course.amount - discount) for Razorpay
           order.
         - Stores original_amount, discount_amount, coupon_code,
           coupon_redemption_id on course_purchases row.
         - On any failure after reserve: release_coupon_reservation.
         - Rate limit: 30/min per IP.
         - 100% off scenario rejected (final ≤ 0 — Razorpay
           mandate).
    4) verify_payment (modified)
         - Reads expected amount from purchase row (not course

         - Payment amount must match purchase.amount exactly.
