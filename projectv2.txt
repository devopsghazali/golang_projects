4. FULL FLOW MAP — TOOLS INVOLVED PER FLOW
----------------------------------------------------------------

  USER BUYS COURSE:
    Browser  → React (ApplyNowModal)
             → fetch (create_order edge fn)
             → Supabase (insert course_purchases, reserve coupon)
             → Razorpay orders.create API
             → Razorpay Checkout (script loaded in browser)
             → User enters card/UPI
             → Razorpay captures payment
             → Browser handler → verify_payment edge fn
             → HMAC signature verify + amount check
             → Supabase (update purchase status=verified,
               confirm_coupon_reservation)
             → Redirect /success with encoded payload
             → User clicks Drive link → Google Drive access

  COUPON APPLY:
    Browser  → CouponInput
             → validate_coupon edge fn (preview, rate-limited)
             → Supabase preview_coupon RPC
             → UI shows discount + remaining slots
             → On purchase, try_reserve_coupon RPC locks seat
               (SELECT FOR UPDATE, 30-min hold)
             → Payment success → confirm_coupon_reservation
               (used_count++)
             → Payment fail/abandon → release after 30 min
               (lazy cleanup) or explicit release call

  ADMIN MANAGES COUPONS:
    Browser  → AdminLogin (token in sessionStorage)
             → adminRequest('list'|'create'|'update'|
                'delete'|'toggle'|'stats')
             → admin_coupons edge fn
             → assertAdmin (safeEqual check)
             → Supabase direct SQL (service role)
             → coupon_audit_log insert per write
             → UI refresh

  PAYMENT WEBHOOK (async backup):
    Razorpay → razorpay_webhook edge fn (HMAC verify)
             → Supabase update purchase (idempotent)
    This is the safety net if user closes browser mid-verify.
